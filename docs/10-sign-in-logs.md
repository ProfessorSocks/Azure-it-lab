# Sign-In Logs

## What Are Sign-In Logs

Sign-in logs in Microsoft Entra ID provide a record of all authentication attempts for users within a tenant.

These logs capture detailed information about each login attempt, including whether it was successful or failed.

## Information Included

Sign-in logs typically include:

- Status (success or failure)
- Failure reason
- Timestamp of the login attempt
- Location or IP address
- Device or application used

## Why Sign-In Logs Are Important

Sign-in logs are a critical troubleshooting tool because they provide direct insight into why a login attempt failed.

Instead of guessing, administrators can identify specific causes such as:

- incorrect credentials
- account being blocked
- multi-factor authentication requirements
- policy-based restrictions

## Troubleshooting Value

Using sign-in logs allows for a structured troubleshooting approach:

1. Identify the failure
2. Read the error message
3. Determine the root cause
4. Apply the correct fix

## IT Support Relevance

Sign-in logs are one of the most valuable tools for diagnosing authentication issues in real-world IT support environments. They help reduce guesswork and improve resolution speed.
