# Environment Setup

## Initial Plan

The original plan was to create a Microsoft 365 Developer sandbox in order to practice Microsoft 365 administration in a lab environment.

## What Happened

I successfully joined the Microsoft 365 Developer Program, but the sandbox creation options were not available. The expected E5 developer subscription and sandbox setup workflow did not appear in the dashboard.

Because of this, I could not proceed with the original Microsoft 365 sandbox plan.

## Pivot

To continue building relevant skills, I switched to Microsoft Azure / Microsoft Entra ID and created or accessed an organizational tenant for practice.

This allowed me to continue working on identity and directory-related tasks, which are highly relevant to Microsoft 365 support.

## Why This Pivot Still Makes Sense

Even without a full Microsoft 365 sandbox, Microsoft Entra ID is still valuable because it covers:

- user identity
- directory structure
- tenant context
- authentication-related troubleshooting
- admin roles and permissions

These are all important for real IT support work.
