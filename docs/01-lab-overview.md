# Lab Overview

## Objective

The objective of this lab is to gain practical experience with Microsoft Entra ID in a way that supports entry-level IT support and help desk job preparation.

## Background

I originally intended to use a Microsoft 365 Developer sandbox, but access to the sandbox environment was not available. As a result, I pivoted to Microsoft Entra ID through Azure in order to focus on identity, users, roles, and troubleshooting.

## Why I Chose This Lab

Identity is one of the most important parts of Microsoft 365 support. Even when the visible issue appears to be related to Teams, Outlook, or general login problems, the root cause often involves the user account, tenant, permissions, or authentication flow.

## Learning Goals

- Understand how Microsoft Entra ID relates to Microsoft 365
- Learn how to navigate the Entra admin center
- Recognize the importance of tenant context
- Access and review users inside the correct directory
- Investigate why admin tasks may not appear even with elevated roles
- Document troubleshooting clearly and professionally

## Career Relevance

This lab is relevant to roles such as:

- IT Support Specialist
- Help Desk Technician
- Desktop Support Technician
- Junior Systems Administrator

These roles often involve identity-related tasks such as account troubleshooting, access verification, password reset workflows, and escalation when admin rights or licensing limitations are involved.

## Resume-Ready Skills

This lab demonstrates hands-on experience with:

- Microsoft Entra ID (Azure Active Directory)
- User account management
- Authentication troubleshooting
- Multi-Factor Authentication (MFA)
- Conditional Access policies
- Sign-in log analysis

The scenarios documented in this repository reflect real-world IT support workflows.
